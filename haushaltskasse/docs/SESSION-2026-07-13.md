## 2. Was du zu Hause tun musst

### 2.1 Aktuellen Stand holen
```
cd <Repo>
git checkout claude/status-update-bpmbn2
git pull
```
Falls neue Pakete fehlen (bcrypt, itsdangerous):
```
pip install -r requirements.txt
```

### 2.2 DB-Schema aktualisieren (einmalig, wegen neuer Spalten)
Die neuen Features nutzen Spalten, die per `ALTER … IF NOT EXISTS` nachgezogen werden.
Gegen deine Azure-DB laufen lassen (Verbindung steht in `.env`):
```
python -m haushaltskasse.storage.db
```
Idempotent — ändert nichts an vorhandenen Daten, ergänzt nur fehlende Spalten.

### 2.3 Login einrichten (P0.1)
```
python -m haushaltskasse.dashboard.auth
```
Fragt dein Passwort ab und gibt vier Zeilen aus → in die **`.env`** kopieren:
```
HAUSHALT_APP_USER=...
HAUSHALT_APP_PASSWORD_HASH=...
HAUSHALT_SESSION_SECRET=...
HAUSHALT_HTTPS_ONLY=1     # nur für den späteren Online-Betrieb; lokal weglassen/0
```
> Ohne `HAUSHALT_APP_PASSWORD_HASH` läuft das Dashboard **ohne** Login weiter (Dev-Modus, mit Warnung).
> Vor dem öffentlichen Deploy ist das Setzen Pflicht.

### 2.4 Dashboard starten
```
python -m haushaltskasse.dashboard.app
```
→ `http://localhost:3000` (im WLAN: `http://<PC-IP>:3000`).

### 2.5 (Später, optional) Online-Deploy
Nach Anleitung `haushaltskasse/docs/DEPLOY.md` — Variante A (App Service Code) zum schnellen
Online-Gehen, Variante C (Container Apps) fürs Sparen. **Vorher** Login-Env setzen (2.3)!

## 3. Was du testen musst

### 3.1 Login (P0.1)
- [ ] Mit gesetzter `.env` das Dashboard aufrufen → **Login-Seite** erscheint.
- [ ] Falsches Passwort → Fehlermeldung. Richtiges → Dashboard.
- [ ] Oben rechts **„Abmelden"** → zurück zur Login-Seite.
- [ ] (Dev) Ohne gesetzten Hash: Dashboard lädt direkt (Warnung im Log) — nur lokal ok.

### 3.2 Buchungen — sortieren & filtern (P1)
- [ ] **Spaltenkopf klicken** (Datum, Betrag, Empfänger, Konto, Kategorie, Unterkat.) → sortiert, ▲/▼ erscheint; nochmal klicken dreht die Richtung.
- [ ] Filter kombinieren: **Datum von–bis**, **Betrag ≥/≤ €**, **Suche**, **Konto/Kategorie/Unterkat.** → „Treffer"-Zahl passt.
- [ ] „zurücksetzen" leert die Filter. URL enthält die Filter (kopierbar, per Reload stabil).
- [ ] Kategorie/Unterkategorie an einer Buchung ändern → „gespeichert", bleibt nach Reload.

### 3.3 Reports — Pivot + Drilldown (P1)
- [ ] Umschalten **Ausgaben / Einnahmen / Netto** und **je Kategorie / je Unterkategorie** → Matrix passt.
- [ ] Zeitraum (von/bis) ändern → Spalten (Monate) passen; **Summenzeile** unten und **Zeilensumme** rechts stimmen.
- [ ] **Doppelklick** auf eine Monats-Zelle → springt in die **Buchungen** gefiltert auf Kategorie × Monat.
- [ ] Doppelklick auf den Zeilen-Namen → Buchungen der ganzen Zeile im Zeitraum.

### 3.4 Rücklagen — Accordion + Fluss (P1)
- [ ] Klick auf eine **Kategorie** klappt die Unterkategorien auf/zu; Zustand bleibt nach Reload (localStorage).
- [ ] „alle auf" / „alle zu" funktioniert.
- [ ] Spalten **Zufluss / Abfluss / Netto** je Kategorie und Unterkategorie sind plausibel;
      Zeitraum (von/bis) ändern wirkt auf die Flüsse (Ist-Topf bleibt).
- [ ] Soll-Beträge editieren → „gespeichert", bleibt nach Reload.

### 3.5 Regressions-Check
- [ ] **Übersicht** und **Config** laden wie gehabt (nichts kaputt).
